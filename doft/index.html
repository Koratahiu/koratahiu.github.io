Exact Orthogonal Weight Decomposition

DOFT: DoRA meets OFT
Korata Hiu
Implementation: OneTrainer PR #1335
Abstract

Weight-Decomposed Low-Rank Adaptation (DoRA) significantly improves over standard LoRA by separating parameter updates into magnitude and direction. However, to calculate the updated directional vectors during the forward pass, standard DoRA implementations must materialize an intermediate full-size weight matrix $\Delta W$, leading to substantial VRAM and compute overhead.

This report details DOFT, a mathematically exact fusion of DoRA with Orthogonal Finetuning (OFT). By exploiting the fundamental property that orthogonal transformations strictly preserve vector norms, DOFT calculates the perfect magnitude scaling without ever materializing modified weights in memory. Through the associative property of matrix multiplication, DOFT shifts an $\mathcal{O}(N_{out} \times N_{in})$ memory bottleneck down to a lightweight $\mathcal{O}(\text{Batch} \times \text{Seq} \times N_{out})$ scaling operation directly on the output activations.

§ 0 Introduction

To understand why DOFT is mathematically elegant, we must look at the mechanics of standard DoRA. In DoRA, a pre-trained weight matrix $W_{orig}$ is modified by applying a learnable magnitude vector $m$ to a normalized directional matrix $V$:

$$W_{DoRA} = m \odot \frac{V}{||V||_c}$$

In standard LoRA-DoRA, $V = W_{orig} + BA$. To calculate $||V||_c$ (the norm of the columns/rows), the system must literally construct $W_{orig} + BA$ in GPU memory during every single forward pass. For large layers (e.g., $8192 \times 8192$), materializing this matrix destroys the memory-efficiency promises of PEFT.

Orthogonal Finetuning (OFT) presents a different paradigm: instead of adding a low-rank matrix, it applies an orthogonal rotation $R$ to the original weights, such that $V = W_{orig} R^T$. The brilliance of combining DoRA with OFT lies in a specific mathematical property of orthogonal matrices: they preserve norms.

§ 1 The Orthogonal Norm-Preservation Property

In OFT, the adapted weights are defined by rotating the base weights using an orthogonal matrix $R$. By definition, for any orthogonal matrix, $R^T R = I$ (the identity matrix).

Let's isolate a single row of the weight matrix, corresponding to a single output channel, denoted as the row vector $w$. We want to find the squared $L_2$ norm of this vector after the OFT rotation is applied:

Squared norm of the rotated vector
$$||w_{oft}||^2 = (w R^T)(w R^T)^T$$
Transpose property: $(AB)^T = B^T A^T$
$$||w_{oft}||^2 = w R^T R w^T$$
Orthogonal property: $R^T R = I$
$$||w_{oft}||^2 = w I w^T$$
Identity reduction
$$||w_{oft}||^2 = w w^T = ||w_{orig}||^2$$
Conclusion: The magnitude (norm) of every single row in the weight matrix is completely unchanged by Orthogonal Finetuning.

§ 2 Deriving the DOFT Optimization

Because we just proved that $||W_{OFT}||_c = ||W_{orig}||_c$, we can substitute the denominator in the standard DoRA equation. We define our direction matrix as $V = W_{OFT} = W_{orig} R^T$:

$$W_{DOFT} = m \odot \frac{W_{OFT}}{||W_{orig}||_c}$$

Notice what happens here: both $m$ (our learnable DoRA magnitude) and $||W_{orig}||_c$ (the frozen initial norm of the pre-trained weights) are static 1D vectors of size [out_features]. We can combine them into a single scaling vector, $S = \frac{m}{||W_{orig}||_c}$.

$$W_{DOFT} = S \odot W_{OFT}$$

The Associative Shift to Output Activations

Normally, a linear layer computes $Y = X \cdot W^T$. If we plug in our materialized DOFT weights, we get:

$$Y = X \cdot (S \odot W_{OFT})^T$$

Because scaling the rows of $W$ by a vector $S$ is mathematically equivalent to scaling the columns of $W^T$, we can use the associative property to pull the scale vector $S$ outside the matrix multiplication entirely:

$$\boxed{\;Y = (X \cdot W_{OFT}^T) \odot S\;}$$

This is the magic of DOFT. The term $(X \cdot W_{OFT}^T)$ is just the standard output of the OFT forward pass. By applying the scale $\odot S$ to the output activations instead of the weights, we achieve identical mathematical results with a fraction of the memory footprint.

§ 3 The Algorithm

By leveraging the associative property, we construct a forward pass that entirely sidesteps weight materialization. We compute the initial norm at startup, and during the forward pass, we simply apply the corresponding scale vector to the output activations returned by the underlying OFT layer.

Algorithm — DOFT Forward Pass
// Initialization (Done once at startup)
W_orig = layer.weight
initial_norm = norm(W_orig, dim=1) ← shape: [out_features]
dora_scale = Parameter(initial_norm.clone()) ← shape: [out_features]

// Forward Pass (Per batch)
def forward(x):
1.y_oft = OFT_forward(x) ← Compute X * (W R^T)^T

2.// DoRA scale
   scale = dora_scale / (initial_norm + eps)

3.y_doft = y_oft * scale.to(y_oft.dtype) ← Apply DoRA scaling to output

return y_doft

§ 4 Memory and Compute Optimizations

The practical consequence of shifting the DoRA computation from the weights to the activations is massive VRAM savings. Materializing $\Delta W$ in standard DoRA requires allocating an $\mathcal{O}(N_{out} \times N_{in})$ tensor, which must be stored in memory for the backward pass to calculate gradients for $m$. DOFT requires zero extra parameter allocations during the forward pass.
